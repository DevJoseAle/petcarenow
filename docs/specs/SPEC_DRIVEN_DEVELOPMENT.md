# Spec Driven Development

Every feature starts with a specification.

## Workflow

```txt
SPEC
↓
PLAN
↓
IMPLEMENTATION
↓
TEST
↓
SUMMARY
```

---

# SPEC

Define:

* Goal
* User Flow
* Screens
* States
* Validations
* Error Handling
* Acceptance Criteria

---

# PLAN

Define:

* Files to create
* Files to modify
* Tests required
* Risks
* Assumptions

No coding yet.

---

# IMPLEMENTATION

Write production-ready code.

Requirements:

* strict typing
* architecture compliance
* reusable code
* no placeholders

---

# TEST

Add or update tests.

Test the layer being modified.

---

# SUMMARY

Document:

* files changed
* tests added
* remaining tasks
* manual verification steps

---
